# 动作编辑模式指令说明

## 📋 指令列表

| 指令名称 | 命令号(十进制) | 命令号(十六进制) | 功能说明 |
|---------|--------------|----------------|---------|
| 进入编辑模式 | 120 | 0x78 | 进入动作编辑模式，舵机失去扭矩 |
| 退出编辑模式 | 121 | 0x79 | 退出动作编辑模式，舵机恢复扭矩 |
| 开始录制 | 122 | 0x7A | 开始录制动作（必须在编辑模式下） |
| 停止录制 | 123 | 0x7B | 停止录制并保存到Flash |
| 播放动作 | 124 | 0x7C | 播放录制的动作（循环播放） |
| 停止播放 | 125 | 0x7D | 停止播放动作 |
| 清除动作 | 126 | 0x7E | 清除录制的动作数据 |
| 查询状态 | 127 | 0x7F | 查询当前状态和帧数 |

## 🔧 协议格式

**发送格式**：`帧头(0xFF) + 命令号`

**示例**：
```
进入编辑模式：发送 0xFF 0x78
开始录制：发送 0xFF 0x7A
停止录制：发送 0xFF 0x7B
播放动作：发送 0xFF 0x7C
退出编辑模式：发送 0xFF 0x79
```

## 📊 详细指令说明

### 1. 进入编辑模式 (CMD_ACTION_EDIT_ENTER = 120 / 0x78)

**发送**：`0xFF 0x78`

**返回**：1字节
- `0x01` - 成功进入编辑模式

**效果**：
- 舵机失去扭矩（可以手动拖动）
- OLED显示：
  ```
  Action Edit
  Mode
  Torque: OFF
  Ready
  ```

---

### 2. 退出编辑模式 (CMD_ACTION_EDIT_EXIT = 121 / 0x79)

**发送**：`0xFF 0x79`

**返回**：1字节
- `0x01` - 成功退出编辑模式

**效果**：
- 舵机恢复扭矩
- 退出动作编辑模式

---

### 3. 开始录制 (CMD_ACTION_EDIT_START = 122 / 0x7A)

**发送**：`0xFF 0x7A`

**返回**：1字节
- `0x01` - 成功开始录制
- `0x00` - 失败（不在编辑模式或已在录制）

**效果**：
- 开始录制动作（每20ms记录一次舵机位置）
- OLED显示：
  ```
  Recording...
  Frames: 0
  (空)
  Recording...
  ```

**注意**：必须先进入编辑模式才能开始录制

---

### 4. 停止录制 (CMD_ACTION_EDIT_STOP = 123 / 0x7B)

**发送**：`0xFF 0x7B`

**返回**：3字节
- 字节0：`0x01` - 成功停止
- 字节1-2：帧数（小端格式）

**效果**：
- 停止录制
- 自动保存到Flash
- OLED显示：
  ```
  Saved!
  Frames: XXX
  (空)
  Ready
  ```

---

### 5. 播放动作 (CMD_ACTION_EDIT_PLAY = 124 / 0x7C)

**发送**：`0xFF 0x7C`

**返回**：3字节
- 字节0：`0x01` - 成功开始播放
- 字节1-2：总帧数（小端格式）
- `0x00` - 失败（不在编辑模式或正在录制）

**效果**：
- 开始播放录制的动作（循环播放）
- OLED显示：
  ```
  Playing...
  Frame: 0/XXX
  (空)
  Playing...
  ```

**注意**：必须先进入编辑模式才能播放

---

### 6. 停止播放 (CMD_ACTION_EDIT_STOP_PLAY = 125 / 0x7D)

**发送**：`0xFF 0x7D`

**返回**：1字节
- `0x01` - 成功停止播放
- `0x00` - 失败（未在播放）

**效果**：
- 停止播放动作
- OLED恢复显示：
  ```
  Action Edit
  Mode
  Torque: OFF
  Ready
  ```

---

### 7. 清除动作 (CMD_ACTION_EDIT_CLEAR = 126 / 0x7E)

**发送**：`0xFF 0x7E`

**返回**：1字节
- `0x01` - 成功清除

**效果**：
- 清除录制的动作数据
- 删除Flash中的动作文件
- OLED显示：
  ```
  Cleared!
  No action
  (空)
  Ready
  ```

---

### 8. 查询状态 (CMD_ACTION_EDIT_STATUS = 127 / 0x7F)

**发送**：`0xFF 0x7F`

**返回**：5字节
- 字节0：编辑模式状态（1=编辑模式，0=正常模式）
- 字节1：录制状态（1=正在录制，0=未录制）
- 字节2：播放状态（1=正在播放，0=未播放）
- 字节3-4：总帧数（小端格式）

**示例返回**：`0x01 0x00 0x00 0xE8 0x03`
- 含义：编辑模式=开启，录制=否，播放=否，帧数=1000

---

## 🎯 完整操作流程示例

### 示例1：录制新动作

```
步骤1：进入编辑模式
发送：0xFF 0x78
返回：0x01
OLED显示：Action Edit Mode / Torque: OFF / Ready

步骤2：开始录制
发送：0xFF 0x7A
返回：0x01
OLED显示：Recording... / Frames: 0

步骤3：手动拖动舵机进行示教（录制中）
OLED实时更新：Frames: 1, 2, 3...

步骤4：停止录制
发送：0xFF 0x7B
返回：0x01 0xE8 0x03  (假设录制了1000帧)
OLED显示：Saved! / Frames: 1000

步骤5：退出编辑模式
发送：0xFF 0x79
返回：0x01
舵机恢复扭矩
```

### 示例2：播放已录制的动作

```
步骤1：进入编辑模式
发送：0xFF 0x78
返回：0x01

步骤2：播放动作
发送：0xFF 0x7C
返回：0x01 0xE8 0x03  (总帧数1000)
OLED显示：Playing... / Frame: 0/1000

步骤3：停止播放（可选）
发送：0xFF 0x7D
返回：0x01
OLED显示：Action Edit Mode / Ready

步骤4：退出编辑模式
发送：0xFF 0x79
返回：0x01
```

### 示例3：清除动作数据

```
步骤1：进入编辑模式
发送：0xFF 0x78
返回：0x01

步骤2：清除动作
发送：0xFF 0x7E
返回：0x01
OLED显示：Cleared! / No action

步骤3：退出编辑模式
发送：0xFF 0x79
返回：0x01
```

### 示例4：查询状态

```
发送：0xFF 0x7F
返回：0x01 0x01 0x00 0x64 0x00
含义：编辑模式=开启，录制=是，播放=否，帧数=100
```

## 💾 技术参数

- **记录间隔**：20ms
- **帧数限制**：无限制（想录多久就录多久）
- **存储位置**：Flash (LittleFS文件系统)
- **存储文件**：`/action_record.bin`
- **自动保存**：每100帧自动保存一次
- **播放模式**：循环播放

---

## ⚠️ 注意事项

1. **必须先进入编辑模式**才能开始录制或播放
2. **录制中不能播放**，播放中不能录制
3. **退出编辑模式会恢复舵机扭矩**
4. **动作数据永久保存**，断电后仍可播放
5. **清除动作不可恢复**，请谨慎操作

## 🔍 串口调试信息

系统会输出以下调试信息：

```
[ActionEdit] Entered edit mode - torque off
[ActionEdit] Recording started
[ActionEdit] Saved 1000 frames to Flash
[ActionEdit] Recording stopped, saved 1000 frames
[ActionEdit] Playback started
[ActionEdit] Playback stopped
[ActionEdit] Action data cleared
[ActionEdit] Status: mode=1 rec=0 play=0 frames=1000
```

---

## 📡 通信方式

支持以下通信方式发送指令：
- **USB串口**（波特率1000000）
- **BLE蓝牙**
- **ESP-NOW无线**
- **WiFi TCP/IP**

所有通信方式使用相同的协议格式。
