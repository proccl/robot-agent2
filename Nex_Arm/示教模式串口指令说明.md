# 示教模式串口指令说明

本固件现在有两个独立玩法，不要混用：

- 无同步器示教：拖动从机本体录制，命令 `120~127`。
- 同步器示教：主机运动，从机只记录 ESP-NOW 主机位置包，停止后再让从机播放，命令 `128~135`。

## 串口帧格式

所有下面的命令都按完整协议帧发送，不是只发一个命令字节。

```text
FF FF ID LEN CMD [ARGS...] CHECKSUM
```

- `ID` 固定用 `FF`
- `LEN` = `CMD + ARGS + CHECKSUM` 的长度；无参数命令固定是 `02`
- `CHECKSUM = ~(ID + LEN + CMD + ARGS...) & 0xFF`
- 波特率：`1000000`

无参数命令示例：开始录制 `CMD=0x7A`

```text
FF FF FF 02 7A 84
```

## 玩法一：无同步器示教

用途：不使用主机同步器，直接拖动从机机械臂录制。从机会关闭舵机扭矩，读取自身位置并保存。

参数：

- 采样间隔：`10ms`
- 最大录制时间：`30s`
- 最大帧数约：`3001` 帧
- 保存文件：`/action_record.bin`
- 录制方式：AT32 每 `10ms` 实时推送 6 个舵机当前位置，ESP32 边录边写 Flash
- 播放结束后保持扭矩打开，停在最后姿势

| 功能 | CMD 十进制 | CMD 十六进制 | 直接发送 HEX |
|---|---:|---:|---|
| 进入无同步器示教 | 120 | 0x78 | `FF FF FF 02 78 86` |
| 退出无同步器示教 | 121 | 0x79 | `FF FF FF 02 79 85` |
| 开始录制 | 122 | 0x7A | `FF FF FF 02 7A 84` |
| 停止录制并保存 | 123 | 0x7B | `FF FF FF 02 7B 83` |
| 播放 | 124 | 0x7C | `FF FF FF 02 7C 82` |
| 停止播放 | 125 | 0x7D | `FF FF FF 02 7D 81` |
| 清空动作 | 126 | 0x7E | `FF FF FF 02 7E 80` |
| 查询状态 | 127 | 0x7F | `FF FF FF 02 7F 7F` |

推荐流程：

```text
清空旧动作       FF FF FF 02 7E 80
进入示教         FF FF FF 02 78 86
开始录制         FF FF FF 02 7A 84
手动拖动从机
停止录制         FF FF FF 02 7B 83
播放             FF FF FF 02 7C 82
退出示教         FF FF FF 02 79 85
```

状态返回数据：

```text
mode recording playing count_l count_h
```

## 玩法二：同步器示教

用途：使用主机同步器。录制时你动主机，从机不运动，只记录主机发来的 ESP-NOW `ArmPacket_t`；停止后，从机按主机原始时间戳回放。

参数：

- 主机发送间隔：按主机当前代码，约 `4ms`
- 最大录制时间：`30s`
- 30s 大约 `7500` 帧
- 保存文件：`/sync_teach.bin`
- 录制方式：ESP-NOW 主机位置包先进 ring buffer，再由主循环写 Flash，不整段占 RAM
- 录制时从机保持扭矩打开但不跟随运动
- 播放时复用实时同步器同一套关节映射和写舵机逻辑

| 功能 | CMD 十进制 | CMD 十六进制 | 直接发送 HEX |
|---|---:|---:|---|
| 进入同步器示教 | 128 | 0x80 | `FF FF FF 02 80 7E` |
| 退出同步器示教 | 129 | 0x81 | `FF FF FF 02 81 7D` |
| 开始录制主机轨迹 | 130 | 0x82 | `FF FF FF 02 82 7C` |
| 停止录制并保存 | 131 | 0x83 | `FF FF FF 02 83 7B` |
| 播放主机轨迹 | 132 | 0x84 | `FF FF FF 02 84 7A` |
| 停止播放 | 133 | 0x85 | `FF FF FF 02 85 79` |
| 清空同步器轨迹 | 134 | 0x86 | `FF FF FF 02 86 78` |
| 查询状态 | 135 | 0x87 | `FF FF FF 02 87 77` |

推荐流程：

```text
清空旧轨迹       FF FF FF 02 86 78
进入同步器示教   FF FF FF 02 80 7E
开始录制         FF FF FF 02 82 7C
手动运动主机，从机此时不动
停止录制         FF FF FF 02 83 7B
播放             FF FF FF 02 84 7A
退出同步器示教   FF FF FF 02 81 7D
```

状态返回数据：

```text
mode recording playing count_l count_h overflow
```

- `overflow=0`：正常
- `overflow=1`：录制缓存来不及写入或空间不足，保存的是不完整轨迹

## 注意

- `120~127` 和 `128~135` 是两个玩法，不要交叉使用。
- 同步器示教录制时不要再发普通实时同步开关命令 `33`。
- 同步器示教前，主机和从机 ESP-NOW 频道要一致。
- 如果录制超过 30 秒，固件会自动停止并保存 30 秒以内的数据。
