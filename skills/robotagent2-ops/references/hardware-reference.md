# robot-agent2 硬件參考 — 幻尔 NexArm

## 品牌型號與定位

- **品牌**：幻尔（Hiwonder）
- **型號**：NexArm
- **定位**：具身智能（Embodied AI）/ 模仿學習（Imitation Learning）開源桌面機械臂
- **兼容生態**：LeRobot / Hugging Face
- **對標產品**：官方宣稱「超越 SO-ARM101」
- **主控板**：ESP32 + AT32F421（雙芯片架構）

## 自由度與關節分布

NexArm 商品頁常標示為「6 軸」，但實際運動學上為 **5 個運動自由度 + 1 個夾爪**：

| 關節 | 對應角度 | 舵機型號 | 說明 |
|------|---------|---------|------|
| 底座旋轉 | θ1 | HX-65HM | 決定機械臂在水平面的方位 |
| 大臂 / 肩部 | θ2 | HX-30HM | 第二連桿 L2 的傾角 |
| 小臂 / 肘部 | θ3 | HX-12H | 第三連桿 L3 的相對傾角 |
| 腕部 pitch | θ4 | HX-10HM | 末端連桿 L4 的相對傾角 |
| 腕部 roll | θ5 | HX-12H | 末端橫滾角 |
| 夾爪 | — | HX-10HM | 開合控制，不計入運動自由度 |

> 注意：腕部 θ4 與夾爪使用 HX-10HM；腕部 roll（θ5）經用戶確認為 HX-12H。下位機會將各關節脈衝統一映射到內部 0–4095 坐標。

## 控制接口維度

`robot-agent2` 與下位機交互的控制維度為：

```python
(x, y, z, pitch, roll, claw)
```

- `x, y, z`：末端位置（單位 mm）
- `pitch`：末端俯仰角（單位度）
- `roll`：末端橫滾角（單位度）
- `claw`：夾爪開合角度（單位度）

**重要**：接口中沒有 `yaw` 控制，因此末端無法繞豎直軸獨立旋轉。這是 5 DOF 手臂的典型特徵。

## 運動學模型

根據 `D:/software package/NexArm机械臂/1.教程资料/3. 运动控制/01 运动控制教程.pdf`，NexArm 的公開教程採用**自定義平面幾何模型**，而不是標準 Denavit-Hartenberg（DH）參數表。

### 連桿參數

| 索引 | 參數 | 物理意義 | 數值 |
|------|------|---------|------|
| [0] | L1 | 底座到肩關節的豎直高度 | 110.45 mm |
| [1] | L2 | 第二連桿主長度 | 225.00 mm |
| [2] | d | 第二連桿末端結構偏移 | 36.97 mm |
| [3] | L3 | 第三連桿長度 | 145.00 mm |
| [4] | — | 連桿 3 偏移 | 0.0 mm |
| [5] | L4 | 末端連桿長度 | 130.23 mm |
| [6] | — | 固定偏移 | 0.0 mm |
| [7] | — | 旋轉半徑（來自用戶補充）| 50.0 mm |
| [8] | base_high | 底座高度 | 70.5 mm |

`arm_kin[9]` 在以下文件中的第 52 行定義：

```
D:/software package/NexArm模仿学习机械臂/1.课程资料/1. 快速入门/02 NexArm出厂程序/Nex_Arm/system_task_handle.cpp
```

具體內容：

```cpp
static float arm_kin[9] = {
    110.45f,   // [0] L1：底座到肩關節的豎直高度
    225.00f,   // [1] L2：第二連桿主長度
     36.97f,   // [2] d：第二連桿末端結構偏移
    145.00f,   // [3] L3：第三連桿長度
      0.0f,    // [4] 連桿 3 偏移
    130.23f,   // [5] L4：末端連桿長度
      0.0f,    // [6] 固定偏移
     50.0f,    // [7] 旋轉半徑（來自用戶補充，源碼中未直接使用）
     70.5f     // [8] base_high：底座高度
};
```

> **關於 [7] 的說明**：當前開源源碼中 `arm_kin` 僅被存儲、讀取和轉發給下位機（`servo.tx_frame_write(0xFF, CMD_SET_KINEMATICS_PARAM, ...)`），實際運動學計算在閉源下位機固件中完成。已對 `01 运动控制教程.pdf` 第 2–9 頁進行視覺檢查，該教程只涉及 L1、L2、d、L3、L4 五個參數，未出現 [7] 或「旋轉半徑」的定義與使用。根據用戶補充，[7] 的含義為「旋轉半徑」，可能與底座旋轉軸偏心、腕部 roll 軸結構或夾爪機構相關，具體幾何作用需結合機械圖紙或詢問廠商進一步確認。

### 角度定義

| 角度 | 定義 |
|------|------|
| θ1 | 底座回轉角 |
| θ2 | L2 相對於水平參考線的夾角 |
| θ3 | 在偏移終點處、相對於與 L2 平行的參考線到 L3 的夾角 |
| θ4 | 從 L3 的延長方向到 L4 的夾角 |
| α | 末端連桿 L4 相對於水平參考線的絕對方向角 |

姿態約束：

```
α = θ2 + θ3 + θ4
```

### 正運動學

1. 先在二維工作平面內求水平伸展量 `length` 與豎直高度 `height`
2. 再通過底座回轉角 θ1 映射回三維空間

工作平面內的位移：

```
length = L2·cosθ2 + d·sinθ2 + L3·cos(θ2+θ3) + L4·cos(θ2+θ3+θ4)
height = L1 + L2·sinθ2 - d·cosθ2 + L3·sin(θ2+θ3) + L4·sin(θ2+θ3+θ4)
```

三維空間映射：

```
x = length · cos(-θ1)
y = length · sin(-θ1)
z = height
```

### 逆運動學

求解流程：

1. **坐標降維**：由 `length = sqrt(x² + y²)` 和 `θ1 = -atan2(y, x)` 分離底座角
2. **腕點回推**：由末端姿態 α 回推腕點 `(a, b)`
3. **等效連桿**：將 `L2 + d` 結構等效為 `L2'` 與修正角 `δ`
4. **三角形約束**：用余弦定理求等效肘角 `γ`
5. **角度恢復**：將等效模型中的角度映射回真實關節角 `θ2, θ3, θ4`

### 關於 DH 參數表的說明

目前從教程 PDF 和出廠程序源碼中**未找到標準 Denavit-Hartenberg（DH）參數表**：

- 教程使用的術語是 L1/L2/d/L3/L4 等幾何長度，而非 DH 的 θ/d/a/α 四參數
- 未出現「DH」、「Denavit-Hartenberg」、「齊次變換矩陣」等關鍵詞
- 角度定義圍繞連桿方向與水平參考線，而非關節軸旋轉

但這並不排除底層實現可能使用 DH 或修改版 DH 進行計算，因為下位機閉源部分或 AT32 固件可能採用不同表示。因此應謹慎表述為「教程層面使用自定義幾何模型，未公開標準 DH 表」。

## 舵機規格與內部脈衝映射

根據各型號總線舵機手冊：

| 型號 | 控制脈衝範圍 | 對應角度 | 理論 pulse/° |
|------|-------------|---------|-------------|
| HX-65HM | 0 – 4095 | 0° – 360° | 11.375 |
| HX-30HM | 0 – 4095 | 0° – 360° | 11.375 |
| HX-10HM | 0 – 4095 | 0° – 360° | 11.375 |
| HX-12H | 0 – 1000 | 0° – 240° | 4.167 |

`get_real_joint_angles()` 回傳的 `pulse` 並非原始舵機脈衝，而是下位機統一映射到 **0–4095** 的內部坐標。以 J5（roll，HX-12H）為例：

- home 時 J5 pulse ≈ 2050（對應 HX-12H 原始中位約 500）
- `roll=+50°` 後 J5 pulse 從 2050 → 2900，變化 **849**
- 實測內部比例 ≈ **16.98 pulse/°**，接近 `4095 / 240 ≈ 17.06`
- 換算回 HX-12H 原始脈衝：`16.98 / 4.096 ≈ 4.15 pulse/°`，與理論 4.167 吻合

## 通信協議摘要

- **接口**：USB 串口（本項目使用 `COM5`）
- **波特率**：1000000
- **數據位**：8
- **校驗位**：None
- **停止位**：1
- **幀格式**：`0xFF 0xFF ID Length CMD Parameter... Check`

常用指令：

| 功能 | CMD | 說明 |
|------|-----|------|
| 坐標設置 | 0x08 | `Pitch + X + Y + Z + Roll + Claw + Time` |
| 當前坐標查詢 | 0x0B | 返回 `X + Y + Z` |
| 坐標增量控制 | 0x32 | `dX + dY + dZ + dPitch + dRoll + dClaw + Time` |
| 運動學參數設置 | 0x3F | 下發 `arm_kin[9]` |
| 運動學參數查詢 | 0x40 | 讀取 `arm_kin[9]` |

## 已知問題與注意事項

### `roll` 指令值與實際角度

經 2026-07-06 實機驗證，`roll` 指令值與實際角度**基本為 1:1**：

| 指令值 | 回傳 J5 angle | 回傳 TCP roll |
|--------|--------------|---------------|
| `roll=+45` | 44.9° | 44.9° |
| `roll=-45` | -44.8° | -44.8° |
| `roll=+50` | 49.9° | 49.9° |

- 先前記錄的「roll=45 實際約 60°（4:3）」應為目測誤差，已修正
- 可放心將 `roll` 指令值視為實際物理角度（單位度）
- 底層 J5 為 HX-12H，下位機內部將其脈衝映射到 0–4095 坐標，換算後與 HX-12H 原始 4.167 pulse/° 吻合

### `claw` 縮放待驗證

`claw` 同樣以指令值下發，但尚未進行精確標定。由於夾爪機構為開合自由度，其「角度」與張開距離的關係需單獨測量。

## 重要結論

1. NexArm 實際為 **5 DOF 手臂 + 夾爪**，不是標準 6 軸工業臂
2. 末端姿態只能控制 `pitch` 與 `roll`，沒有 `yaw`
3. 公開教程使用自定義平面幾何模型講解運動學，未提供標準 DH 參數表
4. 實際控制接口是末端位姿 `(x, y, z, pitch, roll, claw)`，逆運動學求解由下位機完成
5. `arm_kin[9]` 是下位機運動學參數，上層代碼只負責存儲和轉發
6. `roll` 與 `claw` 的指令值與實際物理角度可能存在非 1:1 關係，需實機標定
